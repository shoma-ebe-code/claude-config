## 障害調査

> 思いつきで修正を試さない。4フェーズを順番に実施する。

**フェーズ1 観察** — 何が起きているかをログ・メトリクスで確認する

**フェーズ2 仮説構築** — 原因の候補を列挙する（複数可）
```
仮説A:
仮説B:
仮説C:
```

**フェーズ3 検証** — 仮説を1つずつ検証する。推測で修正しない
```bash
# 仮説Aの検証コマンド例
```

**フェーズ4 修正** — 根本原因が特定できたら最小限の修正を実施する

> ⚠️ 本番環境への変更は必ず2名以上で確認すること

```bash
# 修正コマンド例（実際の環境に合わせて変更すること）
```

**成功の判断基準**:
- [ ] エラーレートが正常値に戻った
- [ ] アラートがクリアされた
